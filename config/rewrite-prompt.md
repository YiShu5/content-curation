### `why_watch`（为什么值得看）

**目标**：写一段让任何人都能立刻抓住价值、又让行家感到"这个角度准"的推荐语。

**结构**（3–4 句，100 字以内）：
1. **第一句**：用最普通的语言，说清楚这期内容讲的是"一件什么事"——不用术语，你妈能听懂
2. **第二句**：点出这件事里最值得停下来想的那个反直觉之处或核心张力
3. **第三/四句**：说明看完之后，读者会带走什么——是一个新框架、一种新眼光，还是一个能直接用的判断

**语气**：朋友之间推荐，不装、不鸡汤、不堆形容词。行家看了觉得"说到点子上了"，小白看了觉得"哦我也能看"。

**禁止**：
- 开头用"这期内容……"、"本期探讨了……"
- 空洞表扬（"非常精彩"、"干货满满"）
- 照抄 core_ideas 的内容

### `key_quotes`（金句）

**目标**：提取 3-5 句能让读者"截图保存"的金句。

**标准**：
- 必须是原文中真实出现的内容（逐字或极高度还原）
- 不得改写、合并或虚构
- 中文内容直接引用，英文内容翻译成地道中文（保留原意，不要直译腔）
- 每条 30-100 字，不得截取零散半句

**选句标准（按优先级）**：
1. 范式级断言：重新定义某个概念或推翻常识的说法
2. 经验结晶：说出了很多人有感觉但从未说清楚的东西
3. 因果洞察：揭示了不明显的 A→B 逻辑链
4. 行动启示：可以直接改变读者行为的建议

**坏金句（禁止选择）**：
- 废话式观点（"努力很重要"、"要持续学习"）
- 信息陈述（"他说他从 XX 大学毕业"）
- 过渡语句（"接下来我们聊聊..."）

### `core_ideas`（核心观点）

**目标**：用 3-5 句话，让读者在 30 秒内理解这期内容的骨架。

**标准**：
- 每条 20-50 字，一句话一个完整论断
- 不是话题列举（"讨论了 AI、谈到了投资"），而是有立场的主张（"AI 不会消灭工作，但会重组工作的价值分配"）
- 各条之间有逻辑递进或互补关系，不得重复
- 如果内容本身没有主张，则提炼内容隐含的逻辑

**测试**：读完这 5 条，能否重建内容的基本框架？如果可以，就是好的。

**结构（请按此结构组织，使用 markdown 标题和格式）**：

**写作要求**：

**禁止**：
- 连续超过 3 个要点的纯列表，要用段落包裹
- 结尾用"综上所述"、"总之"、"总结来说"

## `scores`（内容评分）

**在完成所有其他字段之后，对本期内容进行客观评分。** 评分要有依据，不能全打高分，也不能保守地全打中间值——真正优质的内容应该得高分，平庸的内容应该得低分。

---

### AI 相关性（`ai_relevance`，0-40 分）

评估内容与 AI、科技前沿、数字化转型等议题的相关程度与深度。

| 分段 | 标准 |
|------|------|
| 35-40 | AI/科技是绝对核心，有第一手洞察、数据或独特视角，圈内人必看 |
| 25-34 | AI/科技是重要议题，占内容 50% 以上，有实质性分析 |
| 15-24 | 提及 AI/科技但不是主体，或涉及但缺乏深度 |
| 5-14 | 与 AI/科技关系较弱，偶有涉及 |
| 0-4  | 完全无关，纯商业/人文/生活类内容 |

> 注：AI 相关性低≠内容差，优质的非 AI 内容在其他维度可以拿高分。

---

### 故事性（`storytelling`，0-30 分）

评估内容的叙事质量：是否有引人入胜的结构、具体生动的案例、情感共鸣或戏剧张力。

| 分段 | 标准 |
|------|------|
| 26-30 | 叙事极佳：有清晰主线、具体故事、反转或情感高潮，听完意犹未尽 |
| 20-25 | 叙事良好：结构清晰，有具体案例支撑，不只是观点罗列 |
| 13-19 | 叙事一般：有基本结构，但案例模糊或情节平淡 |
| 6-12  | 叙事较弱：主要是信息堆砌，缺乏连贯性 |
| 0-5   | 几乎无叙事：散乱、重复、难以跟随 |

---

### 加分项（`bonus`，0-30 分）

评估内容是否包含以下高价值元素，**每项独立判断，符合则加分**：

| 加分项 | 分值 | 判断标准 |
|--------|------|----------|
| 数据与研究 | +10 | 引用了具体数字、研究报告、实验结果（不是泛泛而谈） |
| 可操作框架 | +10 | 提供了读者可以直接套用的方法、框架、步骤或清单 |
| 反直觉观点 | +10 | 至少有一个让人"咦，原来如此"的颠覆性论断，且有论据支撑 |

`items` 字段列出实际触发了加分的项目名称（如 `["数据与研究", "反直觉观点"]`），未触发的不列入。

---

### 总分与评级

`total` = `ai_relevance.score` + `storytelling.score` + `bonus.score`（满分 100）

`verdict` 根据总分自动判定：

| 总分 | verdict |
|------|---------|
| 90-100 | 必读 |
| 75-89  | 强烈推荐 |
| 60-74  | 推荐 |
| 45-59  | 一般 |
| 0-44   | 可跳过 |

`reason` 字段（每个维度都需要填写）：用 1-2 句话说明打这个分的具体依据，不能只写"内容不错"这种空话。
