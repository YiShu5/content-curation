# 「降噪 / NoiseFilter」产品文档：用户 · 需求 · 场景

> **本文档用途**：用于和外部 AI / 顾问讨论产品方向。读者无需任何前置上下文。
> 文末列出了**待讨论的开放问题**，欢迎针对性反馈。
> 最后更新：2026-06-19

---

## 0. 一句话产品

> 给 AI 从业者的每日降噪简报：自己快速看懂，也能直接转给同事。

**降噪（NoiseFilter）**：把 YouTube / 播客 / B站 / 小红书上的优质**长内容**（多为英文一手 AI / 科技 / 创投访谈），用 AI 自动转录 → 改写成**结构化中文深度笔记**（标题 / 嘉宾 / 金句 / 核心洞察 / 2000 字摘要 / 质量评分），并沉淀成每日简报与可语义检索、可降噪筛选的**个人知识库 + 内容流**。

口号：**"你不需要看完所有内容，你需要一个降噪系统。"**

---

## 1. 产品现状（已实现，供判断"从哪儿起步"）

这是一个**已经能跑通的真实管线**，不是概念：

**核心流水线**（三段）：
1. **抓取**：从配置的订阅源（YouTube 频道 / B站 UP / 小宇宙播客等）拉新内容元数据（yt-dlp）+ 转录（BibiGPT 主力，baoyu 免费降级兜底）。
2. **AI 改写**（DeepSeek）：输出结构化 JSON —— 中文标题、话题分类、推荐语、嘉宾、3 条金句、深度洞察、1500–2000 字深度摘要、**质量评分**。
3. **归档 + 展示**：本地结构化归档 + 同步飞书多维表格 + 一个"降噪"风格的 Flask 博客。

**已具备的能力**：
- **质量评分体系**（最新版）：洞察原创 0–50 + 信源质量 0–25 + 故事可读 0–25 = 100，自动评级（必读 / 强烈推荐 / 推荐 / 一般 / 可跳过）。**评分只衡量内容质量，与题材无关**（是否 AI 用话题标签区分）。
- **语义搜索**：智谱 embedding-3 建向量索引，按"意思"检索（关键词不命中也能找到）。
- **相关推荐**：详情页基于向量相似度推荐 3 条相关内容。
- **结构化嘉宾卡片**：每位嘉宾的头衔 / 背景 / "本期角色"。
- **每日日报**（规划中，先手动）：把当天新抓内容按题材分两栏（AI 强相关 / AI 引申），只推 ≥推荐 的，生成一份飞书文档。

**内容画像**（实际处理过的）：Lex Fridman（黄仁勋、Jeff Kaplan）、a16z、Y Combinator、Bill Ackman 投资课、AI Skill 教程等 —— **以英文一手前沿 AI / 硬科技 / 创投 / 投资长内容为主**。

**话题分类**：AI 编程 / AI 产品 / AI 创业 / AI 商业 / 投资 / 个人效率 / 其他。

## 2. 用户定位

> **关键前提**：本产品当前形态（单人配置、命令行、私有飞书库、无多用户）决定了它本质是**一个人的内容管线**。下文按"主用户 = 运营者本人"展开，同时标注面向读者的次级用户。

### 主用户（Primary）：靠"持续消费 AI / 科技 / 创投信息"创造价值的知识工作者
典型画像 —— **AI KOL / AI 产品经理 / 独立研究者 / 创投从业者**（运营者本人即此画像，footer 自述 "AI KOL | AI PM"）。

特征：
- 信息源**高度依赖英文长内容**（播客 / 访谈 / Demo），但没时间逐条看完；
- 内容对他是**生产资料**（要写公众号 / 小红书 / 博客 / 做内部分享 / 形成判断），不是消遣；
- 已用飞书做知识管理，有一定动手能力；
- 痛点是"怕错过一手信号"和"看不完"的双重焦虑。

### 次级用户（Secondary）：关注 AI 的中文读者
通过公开的每日简报与深度内容库阅读策展输出，并可把稳定的日期链接继续分享给同事。

## 3. 用户需求（Jobs-To-Be-Done）

**核心痛点**：优质长内容太多、太长、且多为英文 → "怕错过 + 看不完"。

按 JTBD 拆解，主用户雇这个产品来完成四件事：

| # | Job（要完成的任务） | 对应能力 | 现状 |
|---|---|---|---|
| 1 | **降噪筛选**：先告诉我"哪条值得花时间" | 质量评分 / 评级 | ✅ 已有，是最差异化的一环 |
| 2 | **跨语言压缩**：把 1 小时英文榨成 5 分钟结构化中文 | 转录 + 改写 + 摘要 | ✅ 已有 |
| 3 | **可复用沉淀**：变成能搜、能引用的个人知识资产 | 语义搜索 / 相关推荐 / 归档 | ✅ 基本有 |
| 4 | **降低生产门槛**：为"我要输出内容"提供半成品 | 金句 / 观点 / 摘要可直接用 | 🟡 半成品有，"一键成稿"未做 |

**深层需求（目前未满足，是升级方向）**：
- **跨内容综合**：不是单条摘要，而是"本周 5 个人都在谈 X，共识与分歧是什么"、矛盾观点并置、趋势探测。
- **对知识库提问（RAG 对话）**："我攒的内容里关于 Y 有哪些观点？谁说过 Z？"
- **个性化**：系统根据"我读完 / 收藏 / 跳过"的行为，**学我的口味**，动态调整评分与选源。
- **主动性**：帮我发现新信源、主动提醒必读、标记"反共识"内容（防同温层）。

## 4. 用户场景（Day / Week in the life）

1. **个人阅读**：打开最新一期简报，用几分钟看懂当天最多三个事件话题，先获得编辑排序与判断。
2. **深入学习**：对某个话题有兴趣时展开依据，或继续进入现有深度内容库，阅读长摘要、核心洞察与相关内容。
3. **公司分享**：复制当日简报的日期链接，直接发给非技术或技术同事，无需再为不同读者制作第二个版本。
4. **当前边界**：公司专属上下文、内部知识对齐与团队个性化不在当前产品范围内。

简报的产品优势仍然是媒体 / 编辑式的选题、排序与判断，而不是另一套 SaaS 工具。

---

## 5. 差异化与护城河

- **不是"AI 摘要"**（满大街），而是：
  1. **策展品味**：质量评分 + 选源，且（未来）随使用个性化沉淀 —— 别人复制不了；
  2. **跨内容综合 + 观点（POV）** —— 把重复的资讯综合成有判断的洞察；
  3. **细分卡位**："中文 + 深度 + 一手英文 AI 前沿 + 偏好一手信源" —— 一个被巨头忽略的缝隙。
- **护城河本质是"媒体 / 编辑"属性，而非技术**（技术已被商品化）。

## 6. 竞品与市场格局（2026-06 实查）

产品同时踩在三层市场，拥挤度天差地别：

**第 1 层 · 单条音视频总结（转录+摘要）—— 红海，已商品化**
- 国外：Snipd、Podwise、Podurama、NotebookLM、MurmurCast、NoteGPT…
- 国内：**BibiGPT**（本产品的转录依赖，本身是该层 2026 榜首，支持 30+ 平台 + 思维导图 + AI 对话 + 知识库 + 多模型）、通义听悟、小宇宙"问问小宇宙"、Ai好记…
- 大厂内置：字节豆包、腾讯混元 / ima（免费，功能嵌入）。
- ⚠️ **本产品的管线层 ≈ 已被 BibiGPT 等覆盖且做得更多。在这层重做没有壁垒。**

**第 2 层 · 多源去重 + 每日 digest —— 在整合**
- 三大日报：The Rundown AI（200 万+订阅）、TLDR AI、Superhuman AI。
- 去重 / 个性化层：Readless、Summry。
- **暴露的痛点正是机会**：多日报 60–80% 内容重复；且"人工策展但**不个性化**，所有人收到同一份"。

**第 3 层 · 个性化 + 跨内容综合 + 观点 —— 尚未被占满**
- 没有人把"学你口味 + 跨内容综合 + 中文榨干英文前沿 AI + 带视角"做成聚焦产品。这是本产品的真正机会区。

## 7. 战略判断（当前结论，待挑战）

**"可行吗"取决于解读：**

| 解读 | 可行性 | 理由 |
|---|---|---|
| 自己用 | 🟢 高 | 已能跑、贴合自身源与口味、中文前沿 AI 细分无人替代 |
| 卖工具（SaaS） | 🔴 难 | 摘要层商品化，免费大厂 + BibiGPT 做得更多更便宜 |
| 做媒体 / KOL | 🟡 最可行 | 不卖工具，卖"策展出来的视角"；工具是引擎，产品是内容 / 品牌 |

**"为什么没人做成大产品"的真实原因**（均为"做公司"的劝退项，但对"个人做 KOL"多为利好或不相关）：
1. SaaS 经济账不成立（每用户持续成本高 + 对摘要付费意愿低）；
2. 护城河是编辑品味 = 媒体生意（低估值倍数、强依赖个人、不可规模化），VC / 大厂天然回避；
3. 盘子对创业太小、对个人品牌刚好；
4. 综合 / 个性化"做好"难，大家止步于容易的摘要层；
5. 抓取转录再发布存在版权灰色（公司有风险，个人在水面下）。

**唯一对个人也残酷的理由 = 分发 / 获客**：AI 让"造引擎"变便宜，稀缺的是注意力；防御力来自受众 / 品牌，而攒受众最难最慢。

**已确定的方向**：
- **先自用、后产品化**（"产品化" = 做内容 / 媒体，而非卖软件）。
- 升级重点放在第 3 层能力（综合 / 个性化 / 产出闭环），**不在摘要层加功能**（那是给 BibiGPT 打工）。
- 推广走**内容驱动增长**：日报 / 综述本身即可传播的内容；渠道用即刻 / X / 公众号 / 小红书；靠"抢中文首发 + 持续高质量 + 独特声音"冷启动。

## 8. 待讨论的开放问题（希望外部 AI 重点给意见）

1. **定位收敛**：把它当"自用神器"就够，还是值得当成"成为 AI KOL"去熬 1–2 年分发？这是最大的岔路。
2. **真实 TAM**：中文里"要深度提炼英文一手 AI 长内容 + 要观点"的人群到底多大？能否支撑一个个人品牌 / 小生意？
3. **护城河成立性**：在 AI 摘要白菜价的当下，"策展品味 + 跨内容综合 + 细分 POV"真能形成壁垒吗，还是会被大厂的"个性化推荐 + 综合"顺手做掉？
4. **分发可行性**：内容驱动增长在中文 AI 圈现在还跑得通吗？前 1000 铁粉的现实路径是什么？
5. **差异化锐度**：和 Readless（去重+个性化）、BibiGPT（全平台摘要+知识库）、各家 AI 日报相比，"降噪"最不可替代的那一句话价值主张应该是什么？
6. **变现**：若走媒体路线，是订阅 / 付费社群 / 广告 / 引流到咨询，哪条更现实？
7. **风险**：版权灰色、对单一转录与 LLM 供应商的依赖、个人精力不可持续 —— 哪个最可能先杀死它？

---

## 9. 一句话总结

> **"降噪"作为要卖给别人的 AI 工具不可行；作为支撑个人成为"中文 AI 前沿策展 KOL"的私人内容引擎，可行且时机不错——但成败 90% 取决于运营者愿不愿意、能不能熬过"分发 / 获客"这一关，而非技术。**
