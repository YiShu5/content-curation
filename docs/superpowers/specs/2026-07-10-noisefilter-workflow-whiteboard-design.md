# NoiseFilter 工作流飞书画板设计

## 目标

创建一张可编辑的飞书画板，完整呈现 NoiseFilter 当前的内容策展与判断学习工作流。画板用于团队沟通与项目说明，既要让非技术成员看懂内容如何从信号变成知识资产，也要让开发者看清本地归档、向量索引、网站和反馈闭环之间的关系。

## 视觉方向

- 结构：三层水平分区，沿用参考图的“大底板 + 阶段标签 + 主流程箭头”。
- 风格：`Riptide Cobalt`，奶油色画布、深海军蓝模块、钴蓝连接线。
- 密度：中等；主链路突出，技术细节仅保留到系统组件级，不展示函数名和文件路径。
- 尺寸：约 1680 × 1180 的逻辑坐标空间，由内容边界决定最终画板大小。
- 形状：只使用圆角矩形、圆形、直线、直角连接线和文本，确保导入飞书后保持可编辑。

## 信息架构

### 1. 信号获取

入口分为两类：

- 自动入口：YouTube、Bilibili、小宇宙订阅源，由每日任务扫描新内容。
- 手动入口：用户提交单条 URL，或从“今日关注”中选择建议内容入库。

主流程为：

`订阅源 / URL → 元数据抓取 → 转录获取 → 原始内容包`

转录节点标注两级策略：BibiGPT 主路径，以及 YouTube 免费转录兜底。

### 2. AI 加工与知识沉淀

主流程为：

`原始内容包 → AI 改写与三维评分 → 本地 archive 真相源 → 内容补全 → 向量索引`

其中：

- AI 改写输出中文标题、嘉宾、金句、核心观点、关键洞察和深度摘要。
- 三维评分包括洞察原创、信源质量、故事可读。
- `archive` 保存 metadata、transcript、rewritten 和 cover，是网站的唯一当前真相源。
- 内容补全包括嘉宾介绍和精选金句。
- 向量索引服务于语义搜索、相关推荐和今日判断。

飞书多维表格与飞书文档是从 `archive` 出发的可选同步出口，以虚线连接，避免被误解为网站的数据源。

### 3. 产品呈现与判断学习

网站直接读取本地 `archive`，提供：

- 首页内容卡片
- 文章详情
- 语义搜索与问答
- 相关推荐
- 今日必读与今日关注

用户行为包括点击、采纳、忽略和建议入库。行为被记录后进入判断学习层，影响下一轮今日信号筛选；“建议入库”还会回到手动入口，形成内容生产闭环。

## 连接关系

- 主链路使用粗钴蓝实线箭头，从左向右阅读。
- 自动每日任务使用顶部细线串联“抓取 → 补全 → 索引 → 今日判断”。
- 飞书同步使用虚线，表达其可选性。
- 用户行为到判断学习使用回环箭头。
- “建议入库”从反馈区回到手动 URL 入口，形成第二条回环。

## 画板布局

- 左侧竖排三个阶段标签：`信号获取`、`知识沉淀`、`产品呈现`。
- 上层占画板约 28%，放入口、抓取和转录。
- 中层占约 34%，放 AI 改写、archive、补全、索引和可选飞书同步。
- 下层占约 38%，放网站能力、用户行为和反馈闭环。
- 右上角放标题 `降噪 NoiseFilter`，副标题 `内容策展与判断学习工作流`。
- 不在画板中出现提示词、文件路径、工具说明、设计风格名或来源说明。

## 验收标准

- 非技术成员能沿主箭头在一分钟内说明完整内容流。
- `archive` 被明确标记为当前真相源，飞书仅是可选同步出口。
- 今日判断与用户反馈形成可见闭环。
- 所有文本均在节点内，无溢出、重叠和裁切。
- SVG 只使用飞书支持的原生形状；箭头由连接线的 `marker-end` 生成。
- SVG 检查通过，并在本地渲染图和飞书实时画板中分别完成一次视觉校验。

## 非目标

- 不展示 API 密钥、环境变量或权限配置。
- 不展开每个脚本的函数级调用关系。
- 不把旧版“飞书作为博客数据源”的流程画入当前主链路。
- 不设计项目未来路线图或部署架构。
