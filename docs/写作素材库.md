# 写作素材库 · 范文/风格库（"金句库"）

> 用途:收集"低 AI 味、有人味"的范文片段 + 拆解手法,写周报时借鉴。
> 纪律:**供风格学习,不照搬原文**(范例均注明出处)。配合 vibe-writing skill 用。

---

## 范例 #1 — 公众号 AI 文章开头(woshipm《Anthropic 产品负责人 24 条建议》)

**出处**:woshipm.com/ai/6417230.html(Kat Woo / Lenny's Podcast 笔记体)
**用户点评**:"一看也是 AI 写的,但比较有人味。"

原文开头(摘录,仅供学习):

> 上周末刷到 Lenny's Podcast 更新了一期新访谈,嘉宾是 Anthropic 的 Claude Coding 产品负责人 Kat Woo。
> 我本来只想随便听两分钟就去忙别的,结果一个半小时听完了,中间暂停了七八次做笔记。
> 这期含金量太高了。
> Kat 不是那种站在台上讲「我们的愿景是让世界更美好」的产品经理。她分享的几乎全是实操层面的东西……
> 我听完把笔记整理了一遍,发现里面的内容对这 3 类人特别有价值……
> 可能有些想法还不成熟,但我已经尽量把最有信息量的部分一条条拎出来了,总共 24 条。
> 坦率地讲这篇更像是我的学习笔记,如果刚好对你有启发,那就赚到了。

### 为什么"AI 底子却有人味"——可复用手法

1. **真实时间+平台锚点**:"上周末刷到 Lenny's Podcast" —— 具体到时间和来源,不空泛("近日 / 随着 AI 发展"是 AI 套话,这里完全没有)。
2. **反预期 + 行为细节**:"本来只想听两分钟……结果一个半小时,中间暂停七八次做笔记" —— 带数字的小动作,有画面、可信,像真人。
3. **短句直给判断**:"这期含金量太高了。" 独立成段,不绕。
4. **立靶子定义价值**:"不是那种站台上讲『让世界更美好』的 PM" —— 用对立面反衬,比正面夸更有力。
5. **明确写给谁**:点名"这 3 类人" —— 让读者对号入座,觉得"说的就是我"。
6. **自谦去爹味**:"想法还不成熟""更像我的学习笔记""对你有启发就赚到了" —— 降姿态、拉近距离,这是最关键的人味来源。

**一句话**:人味 ≠ 辞藻,而是 **具体细节 + 第一人称真实反应 + 自谦 + 给读者定位**。我们的《本周 AI 必读》开头已经用了 1/3/5,下期可加强 2(行为细节)和 6(自谦)。

## 范文 #2 · 格式模板 —— 单源 N 条干货(woshipm《Anthropic 24 条建议》)

> 用法:挑**一个**信息源(一期播客 / 一个长视频 / 一次访谈),啃透,拆成 N 条编号干货,每条 = 一个观点。这就是"写作降噪":**一篇一个源、一条一个概念**。出处同范文 #1,供格式学习。

**开头公式(约 4 段)**
1. 场景钩子:最近刷到 XX,本来只想随便看看,结果看完了,中间暂停 N 次做笔记
2. 背书短句:"这期含金量太高了"(独立成段)
3. 人物速写:"XX 不是那种站台上讲愿景的人,他/她分享的几乎全是实操……"
4. 三类受众:"整理完发现对这 3 类人特别有价值。一类是…一类是…还有一类是…"
   过渡:自谦 + 数量预告 —— "更像我的学习笔记,总共 N 条"

**主体(每条)**
- 小标题 = `序号. 一个观点句`(陈述/论断,**不用"如何/为什么"提问式**),12-25 字,常带逗号双分句("X 不是 A,而是 B")
- 标题下 3-6 段、每段 1-4 行:① 嘉宾原话/数据 → ② 解释 → ③ 一个案例 → ④「我」的感受(独立成段)
- 口语词撑节奏:对你没听错 / 你敢信 / 坦率讲 / 至少 / 反而;术语保留英文(PM、eval、agent、demo、aha moment)
- 每 4 条插一句承接:"说到 XX,顺着聊聊……"

**结尾公式**
- "说了 N 条,回头看其实有一条暗线贯穿始终" → **加粗·暗线段**(旧范式 vs 新范式) → **加粗·新法则段**
- 自谦句 + 反问(值得隔段时间问自己一次)+ 一句祝愿式金句

**量级**:5000-6000 字 / N≈15-24 条;配图每 4-6 条插 1 张,开头钩子后必插 1 张。
**降噪版调整(给我们用)**:别硬凑条数(宁缺毋滥),**8-12 条精的 > 24 条注水**;每条结尾带一句「对 AI 产品人意味着什么」。

## 可直接套用的句式 / 金句

> 看到好用的开头句、过渡句、收尾金句,丢这儿,下次改着用。

**1. 「写给哪几类人」句式**(出处:woshipm Kat Woo 文)

> 发现里面的内容对这 3 类人特别有价值。一类是正在做 AI 产品的产品经理,不管是在职的、待业的还是准备转行的。一类是 AI Native 方向的创业者。还有一类是自己动手搞产品的个人开发者。

为什么好:① 具体到「在职 / 待业 / 转行」,读者一秒对号入座;② 三类并列、每类一句,干脆;③ 放开头 = 先圈定「这篇就是写给你的」。
怎么改:把三类换成你这篇真正的受众,保留「一类是…一类是…还有一类是…」的节奏。
